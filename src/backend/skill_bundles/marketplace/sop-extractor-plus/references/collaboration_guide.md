# 多人/跨部门协同 SOP 引导模板

适用场景：需要多个角色参与、涉及跨部门交接的流程提取。

---

## 为什么协同场景需要特殊处理

单人线性流程可以直接提取，但跨部门协同流程有以下挑战：
- **角色边界不清**：谁做什么、谁审批、谁接收容易混淆
- **交接点遗漏**：部门间的 handoff 节点常被口述者跳过
- **子流程嵌套**：每个部门内部的子流程可能同样复杂

推荐策略：**先拆分，再分别提取，最后整合**。

---

## 第一步：角色识别（在对话中引导）

触发协同引导模式的 AI 话术：

```
我看到这个流程涉及多个角色/部门，为了让SOP更准确，我先整理一下参与方：

请确认：
1. 这个流程涉及哪些角色？（如：销售、运营、技术、客服）
2. 每个角色负责流程的哪个阶段？
3. 哪些节点需要角色A把任务交给角色B？（这些是交接点）
```

---

## 第二步：分角色描述框架

让用户按角色分别描述，使用以下模板提示：

```
好，我们按角色拆分来描述：

【[角色A]的操作步骤】
1. 触发条件是什么？（什么情况下A开始工作）
2. A具体做什么操作？
3. A做完后，给谁、交什么？（交接点）

【[角色B]的操作步骤】
1. B收到什么信号/材料才开始工作？
2. B具体做什么操作？
3. B做完后，下一步是什么？
```

---

## 第三步：交接点模板

每个跨部门交接点，建议补充：

```
交接点 [编号]：
- 上游角色：[谁]  完成后的输出物：[什么材料/状态]
- 下游角色：[谁]  接收后的触发条件：[什么条件下开始]
- 交接方式：[邮件/企业微信/系统通知/口头]
- 预期时间：[交接应在多久内完成]
- 异常处理：[如果没按时交接，怎么办]
```

---

## 第四步：分批提取命令

```bash
# 提取角色A的子流程
python sop_extractor.py -f roleA_flow.txt -o output/sop_roleA.json

# 提取角色B的子流程
python sop_extractor.py -f roleB_flow.txt -o output/sop_roleB.json

# 分别生成各角色的SOP文档
python sop_formatter.py -i output/sop_roleA.json -f markdown -o output/sop_roleA.md
python sop_formatter.py -i output/sop_roleB.json -f markdown -o output/sop_roleB.md
```

---

## 第五步：整合提示

提取完各角色子流程后，输出整合后的主流程 SOP 框架：

```markdown
# [流程名称] 协同SOP

## 总体流程概览
[泳道图或角色流程概述]

## 角色A — [职责描述]
[从 sop_roleA.md 合并]

## 交接点 1：A → B
- 输出物：___
- 交接方式：___
- 预期时长：___

## 角色B — [职责描述]
[从 sop_roleB.md 合并]

## 交接点 2：B → C（如有）
...
```

---

## 典型案例：内容发布协同流程

**涉及角色**：内容编辑 → 设计师 → 审核 → 运营

| 交接点 | 触发条件 | 输出物 | 交接方式 |
|--------|---------|--------|---------|
| 编辑→设计 | 文字稿定稿 | Word初稿 + 配图要求说明 | 腾讯文档链接@设计师 |
| 设计→审核 | 视觉稿完成 | 设计稿PDF + 源文件链接 | 企业微信发送 |
| 审核→运营 | 审核通过 | 审核意见单（无问题则直接标注"通过"） | 工单系统流转 |

---

## 使用限制提醒

- **本 Skill 目前不能自动合并多个子流程**，整合步骤需人工操作
- 超过5个角色的大型协同流程，建议先画手工草图再提取
- 协同SOP中的异常处理路径尤为重要，务必在追问中专门问清楚
