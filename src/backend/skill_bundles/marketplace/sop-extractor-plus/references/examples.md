# SOP 提取器 — 真实使用案例

共4个完整案例，每个案例包含：触发语 → AI处理过程 → 输出预览。

---

## 案例 1：客服处理流程（标准输入）

**触发语：**
> "帮我把这个流程整理成SOP：收到客户投诉 → 先道歉，然后问清楚是什么问题 → 如果是产品质量问题就走退换货流程 → 如果是操作问题就给教程 → 处理完记录到工单系统 → 24小时内回访确认是否解决"

**AI 处理过程：**
1. 识别 6 个步骤，1 个决策点（质量问题 vs 操作问题）
2. 检测到缺失项：前置条件（客服权限要求）、时间估算、工单系统具体操作说明
3. 质量评分：78/100（B级 — 良好）
4. 输出理解确认行：`📌 我理解你要整理的是「客服投诉处理流程」，共识别到约 6 个步骤，覆盖接单→判断→处理→记录→回访。按【确认】继续，或告诉我理解有误的地方。`

**输出预览（Markdown 格式）：**

```markdown
# 客服投诉处理标准流程

**版本：** v1.0 | **适用岗位：** 客服专员 | **预估总耗时：** 30-60分钟

## 前置准备
- [ ] 登录客服系统，确认工单权限
- [ ] 准备退换货申请模板

## 步骤 1：接单确认（约5分钟）
**操作：** 接收投诉，记录客户信息（姓名/订单号/联系方式）
**工具：** 客服工单系统
**注意：** 先向客户表达歉意，稳定情绪

## 步骤 2：问题诊断（约10分钟）
**操作：** 询问问题详情，判断类型
**决策点：**
  - 产品质量问题 → 执行退换货流程（步骤3a）
  - 操作使用问题 → 提供教程（步骤3b）
...
```

**适用场景：** 标准客服 SOP 沉淀、新人入职培训材料

## 案例 2：简短输入（触发追问）

**触发语：**
> "帮我整理一下发微信公众号的流程"

**AI 处理过程：**
1. 检测到输入过短（< 50字），仅含1个笼统描述
2. **不直接报错**，而是启动追问：
   ```
   你的描述已收到，但缺少关键细节，能补充一下吗？
   1. 内容审核由谁来做（自审 or 编辑审？）
   2. 发布前需要哪些检查（排版/链接/配图？）
   3. 发布后需要做什么（打卡/数据追踪/转发到群？）
   ```
3. 用户补充后，重新提取，识别出 8 个步骤
4. 质量评分：82/100（B级）

**追问价值：** 通过3个追问，步骤数从1个提升到8个，覆盖率从20%提升到90%

## 案例 3：已有旧文档的结构化升级

**触发语：**
> "我有一份旧的部署文档，但写得很乱，帮我升级成标准SOP"（附上500字旧文档）

**AI 处理过程：**
1. 解析旧文档，识别混乱描述中的核心步骤
2. 发现：步骤顺序有误（步骤4实际应在步骤2前执行）、缺少回滚方案、缺少版本号记录
3. 缺口分析：5项缺失，其中2项高优先级（回滚方案、操作人记录）
4. 质量评分：旧版 52/100（D级）→ 优化后 88/100（A级）

**输出：**
- 优化后 SOP（Markdown）
- 新旧对比说明（标注了哪些步骤被重排/补充）
- 执行检查清单（可打印版）

## 案例 4：批量多格式输出

**触发语：**
> "帮我把这个新人入职培训流程整理成SOP，要HTML页面、检查清单和流程图，同时生成综合分析报告"

**AI 处理过程：**
1. 提取 12 个步骤，质量评分 91/100（A级）
2. 运行一键脚本：`python run_all.py -t "..." -o output/ -f all`
3. 生成全部格式

**输出清单：**
```
output/
├── sop.html          # 交互式网页（深色主题）
├── sop.md            # Markdown 归档版
├── checklist.html    # 可打印执行清单
├── flowchart.mmd     # Mermaid 流程图源码
├── training_card.html # 新人培训卡片
└── sop_report.html   # 综合分析报告（含质量评分、缺口分析、优化建议）
```

**一键运行命令：**
```bash
python run_all.py -t "新人入职第一天：签合同、配电脑、加企业微信、参观办公区、分配工位、配权限、午饭认识同事、下午读文档..." -o output/ -f all
```
