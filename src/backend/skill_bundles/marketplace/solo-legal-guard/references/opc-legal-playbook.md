# 一人公司法务风控手册

## 核心原则

一人公司的法务目标不是把合同写得最复杂，而是保护现金流、限制交付风险、保留维权证据、避免股东个人承担公司债务。

## 高频风险

| 风险 | 典型表现 | 处理方式 |
|---|---|---|
| 收款风险 | 验收后一次性付款、客户可无限拖延验收 | 预付款 + 里程碑 + 默认验收通过 |
| 无限修改 | “直到甲方满意为止” | 明确修改次数、需求边界、变更报价 |
| 知识产权 | 只写交付成果，不写著作权和源文件 | 明确归属、转移节点、源文件范围 |
| 维护义务 | “免费维护一年”但无范围 | 限定缺陷修复，不含新增需求和第三方变化 |
| 赔偿过大 | 无责任上限、延期每天1% | 设置累计上限，日万分之五至日千分之一更稳妥 |
| 劳动关系误判 | 长期兼职、每日考勤、固定工资 | 外包按成果结算，避免直接管理 |
| 人格混同 | 个人账户收公司款、公私报销混乱 | 独立账户、合同主体一致、借款协议和凭证 |

## 签约前证据清单

1. 合同正文。
2. 报价单。
3. 需求说明书。
4. 交付清单。
5. 验收标准。
6. 付款节点。
7. 双方联系人和送达地址。
8. 沟通记录、会议纪要、确认邮件。
9. 第三方素材、字体、开源组件授权证明。
10. 发票类型和税费承担约定。

## 五套件协同

| 场景 | 建议联动 |
|---|---|
| 商单来自自媒体账号 | 自媒体矩阵管理器：核对商单报价、交付内容、排期和数据承诺 |
| 需要带货或产品销售 | 电商运营助手：核对选品、售后、退换货、广告合规 |
| 涉及收入和税费 | 财务管家：核对发票、税率、利润和现金流 |
| 商业模式不清 | 创业思维导师：重新判断客户、产品和交易结构 |

## 常用谈判话术

- “为了保证项目能按时推进，我们建议把付款拆成预付款、阶段款和验收款，这样双方都有确定性。”
- “免费维护可以覆盖已有功能缺陷修复，但新增需求和第三方接口变化需要另行评估工期和费用。”
- “验收需要有明确期限，否则项目会无法收尾。我们建议提交后5个工作日内反馈，逾期视为通过。”
- “源代码可以交付，但交付范围、依赖库、部署文档和尾款支付节点需要同步明确。”
