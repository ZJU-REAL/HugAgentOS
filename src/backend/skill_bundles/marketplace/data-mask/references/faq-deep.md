# 数据脱敏深度 FAQ

> 本文档覆盖边缘场景、安全合规、平台兼容等进阶问题。
> 主文档 FAQ 已覆盖常见错误排障（8 题），本文档从第 9 题开始继续。

---

## 第 9 题：脱敏后的数据发给大模型，大模型平台会不会存储我的原始数据？

不会。脱敏后的数据已不包含原始真实数字，即使平台存储数据，也只是脱敏后的替换值。但仍需注意：

- **hash 策略**：同一原始数字脱敏后每次一致，如果多次发送同一脱敏文件，平台可能关联识别
- **placeholder 策略**：数值完全隐藏为标签，最安全
- **建议**：敏感度极高的数据建议用 placeholder + 不保存映射表的方式脱敏

## 第 10 题：脱敏后单位会不会丢失或不匹配？

不会。工具已内置识别中文单位后缀（万元、亿美元、人次等），脱敏时将数字部分替换、单位保持不变。例如：

- `158000万元` → hash → `443350万元`（单位「万元」保留）
- `23.5%` → hash → `77%`（百分号保留）

如果遇到工具未识别的自定义单位，可在 `config.yaml` 的 `custom_units` 中追加。

## 第 11 题：脱敏后的数字太大或太小，会不会溢出 Excel 单元格？

一般不会。但需注意：

- hash 和 blur 策略在量级区间内生成，脱敏后数字量级与原数据相近
- **超大整数（超过 10 位）** 可能因浮点精度产生微小偏差——这是 Python 浮点数精度限制
- 金额类数据建议使用 `.xlsx` 格式，避免 CSV 中科学计数法问题
- 如果遇到 Excel 单元格显示 `####`，拉宽列宽即可，数据本身没有问题

## 第 12 题：多个文件批量脱敏时，同一数字在不同文件中脱敏结果是否一致？

**取决于策略和映射表：**

- **hash 策略 + 同一映射表**：同一数字在不同文件中脱敏结果一致 ✅
- **hash 策略 + 不同映射表**：同一数字在不同文件中脱敏结果不同 ❌
- **blur / range 策略**：每次脱敏结果都不同（随机性）

**批量处理建议**：如果需要跨文件保持一致性，用一个统一映射表：
```bash
# 第1个文件：创建映射表
python scripts/file_mask.py -i file1.docx -m common_map.json
# 后续文件：使用同一映射表
python scripts/file_mask.py -i file2.pptx -m common_map.json
```

## 第 13 题：我需要对 Word 文档中的部分表格脱敏，其他内容保留原样，能做到吗？

目前工具对 Word/PPT 文件做「全文脱敏」，即文档中所有出现数字的地方都会替换。

**替代方案：**
- 将 Word 中需要脱敏的表格复制到 Excel → 用 `dataframe_mask.py` 精准控制列 → 脱敏后的数据复制回 Word
- 或者在脱敏前将不需要脱敏的段落中的数字临时改为非数字格式（如「100万」→「一百万元」），脱敏后再改回

## 第 14 题：脱敏工具和「加密」有什么区别？我需要的是加密不是脱敏怎么办？

| | 脱敏 (Data Masking) | 加密 (Encryption) |
|---|---|---|
| **目的** | 隐藏真实数据但保留可用性 | 保护数据不被未授权访问 |
| **可还原** | hash/placeholder 策略可通过映射表还原 | 需要密钥解密 |
| **数据分析** | 脱敏后可做分析（模糊查询等） | 加密后数据完全不可用 |
| **适用场景** | 数据外发、AI 分析、测试数据 | 存储安全、传输安全 |

**本工具是脱敏工具，不是加密工具**。如果需要加密，请使用专门的加密工具。

## 第 15 题：Windows 和 macOS 上使用时，文件路径包含中文会不会有问题？

- **Windows**：如果终端默认编码不是 UTF-8，中文路径可能导致「文件不存在」错误。解决方法：在终端执行 `chcp 65001` 切换到 UTF-8 编码后再运行
- **macOS / Linux**：中文路径一般无问题，但如果路径中有特殊字符（空格、括号），需加引号：`python scripts/file_mask.py -i "路径/文件.docx"`

---

## 第 16 题：脱敏后生成的映射表文件安全吗？需要怎么保管？

映射表是 JSON 明文文件，包含原始数字到脱敏数字的对应关系。**映射表就是还原密钥**：

- ✅ 单独保管，不要和脱敏文件一起发送
- ✅ 使用后及时删除（如果不需要还原）
- ⚠️ 不要上传到公开的 Git 仓库或云盘
- ⚠️ 如果映射表泄露，脱敏相当于失效

## 第 17 题：我可以把 data-mask 集成到自己的 CI/CD 流程中吗？

可以。所有脚本都是标准 Python CLI 工具，支持命令行参数和编程调用：

```python
# 编程调用示例
from file_mask import mask_document
mask_document('report.docx', None, strategy='hash')

from dataframe_mask import mask_dataframe
import pandas as pd
df = pd.read_excel('data.xlsx')
masked_df, mapping = mask_dataframe(df, strategy='blur')
```

**注意事项：**
- Python 3.8+ 环境
- 按需安装依赖（python-docx, python-pptx, openpyxl 等）
- 脚本非线程安全，多进程场景需各自操作独立文件
