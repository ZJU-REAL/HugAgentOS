# 数据脱敏反模式案例集

> 本文档收录数据脱敏工具最常被误用的场景，每类附「错误示例 ❌ → 正确示例 ✅」对比。
> 使用本工具前阅读本文档，可避免最常见的踩坑。

---

## 反模式 1：用 placeholder 脱敏后发给 AI 做数据分析

这是最高频的误用场景。placeholder 策略将数字替换为 `[A]` `[B]` 等标签，数值的语义信息完全丢失，AI 无法进行任何数值分析。

```
❌ 错误做法：
用户：把这份销售月报脱敏发给AI分析，帮我写个总结
→ 使用 placeholder 策略脱敏
→ 脱敏后：各渠道"订单量 [A]、[B]、[C]"  GAIC 链接"成本 [D]、[E]、[F]"
→ AI 只能给出"需要注意数据"的泛泛总结，毫无分析价值

✅ 正确做法：
用户：把这份销售月报脱敏发给 AI 分析
→ 使用 hash 策略（默认），数字在量级区间内替换但保持相对大小关系
→ AI 可以分析"渠道A订单量最高，渠道C成本最低"等趋势结论
→ 事后还能用映射表还原为真实数字
```

**判断依据：** 数据分析场景必须用 hash 或 blur 策略，placeholder 仅用于"彻底隐藏数值"的场景。

## 反模式 2：忘记保存映射表，事后无法还原

脱敏完成后发现需要还原数据查看原始值，但当时没有保存映射表。

```
❌ 错误做法：
python scripts/file_mask.py -i report.docx
# 没有加 -m mapping.json，映射表丢失
# 一周后领导要原始数据时，只能重新跑一遍

✅ 正确做法：
python scripts/file_mask.py -i report.docx -m report_mapping.json
# 映射表保存为 JSON 文件，随时可还原
# ⚠️ 注意：只有 hash 和 placeholder 支持还原，blur 和 range 不可逆
```

**防护规则：** 每次脱敏都加 `-m` 参数保存映射表，养成习惯。

## 反模式 3：混淆不同策略的特性，期望矛盾的结果

同时期望「看起来像真实数据」和「完全不可还原」，但选择了不支持该组合的策略。

```
❌ 错误期望：
用户选 blur 策略，期望脱敏后数据看起来真实，同时又期望能还原
→ blur 策略不可逆，无法还原

✅ 正确理解：
- 想要「看起来真实 + 可还原」→ 选 hash（默认）
- 想要「看起来真实 + 不关心还原」→ 选 blur
- 想要「完全隐藏数值 + 可还原」→ 选 placeholder
- 想要「知道范围 + 不关心精确值」→ 选 range
```

## 反模式 4：对 PDF 排版期望过高

PDF 脱敏后保留的是文字内容和分页，**不保留原排版样式**（字体、颜色、表格格式、图片）。

```
❌ 错误期望：
用户：我要把这份带图表的 PDF 报告脱敏，排版要和原来一模一样
→ data-mask 的 PDF 输出只保留纯文本+分页，排版会变化

✅ 正确认知：
- 如果必须保留排版：将 PDF 转为 Word（.docx），脱敏后再转回 PDF
- 如果只关心文字内容：直接用 PDF 脱敏，结果已包含所有文字信息
- 如果 PDF 是扫描版：工具自动委托 PDF skill 做 OCR，输出为 .txt
```

## 反模式 5：不确认就对外发送脱敏数据

工具只替换数字，不做语义级安全检查。脱敏后的数据中可能仍包含敏感信息（人名、地名、特定表述等）。

```
❌ 错误做法：
脱敏后直接发给外部第三方，未人工复核
→ 假设原文中有"张经理负责深圳区域的KA大客户"，脱敏只替换了数字
→ "深圳"、"KA大客户"可能仍是敏感信息

✅ 正确做法：
1. 脱敏完成后人工通读一遍输出文件
2. 确认没有残留的敏感非数字信息
3. 工具末尾已提示："脱敏后需人工复核"
```

## 禁忌用法清单

| 禁忌行为 | 为什么不行 | 解决方案 |
|---------|---------|---------|
| 数据分析用 placeholder | 数值语义完全丢失，AI 无法分析 | 数据分析用 hash 或 blur |
| 脱敏不保存映射表 | 事后无法还原原始数据 | 每次加 `-m mapping.json` |
| 期望 blur 策略可还原 | blur 的随机偏移不可逆 | 需要还原请用 hash 或 placeholder |
| 对 .doc/.xls 旧格式直接脱敏 | 旧版 Office 格式不支持 | 先手动转为 .docx/.xlsx |
| 不安装 Tesseract 就要 OCR 图片 | OCR 功能依赖外部引擎 | 按文档指引安装 Tesseract |
| 脱敏后不人工复核就对外发送 | 非数字敏感信息可能残留 | 脱敏后通读确认 |
