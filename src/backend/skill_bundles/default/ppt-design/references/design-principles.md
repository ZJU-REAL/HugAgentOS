# Design Principles —— 为什么这样做

版式表告诉你"做什么"，这份告诉你"为什么"。理解了"为什么"，遇到表里没覆盖的
情况你也能自己判断。一份 deck 显不显业余，九成取决于下面这几条原则。

## 1 · 叙事弧：deck 是一条线，不是一摞卡片

观众是线性接收的。一份好 deck 有起承转合：

```
cover（定调）→ toc（路线图）
  → section 1 → 若干 content（每页讲清一件事）
  → 喘息页（quote / big_number）
  → section 2 → 若干 content …
  → summary（收束，给观众带走的三句话）
```

**为什么要 section 分段**：没有 section 的 deck 像没有章节的书，观众会迷路。
每个 section 是一次"我们现在讲第几部分"的重新定位，认知成本骤降。

**为什么 cover 要有 tagline**：一句话定调（"—— 用数据重新认识这个行业"）让
观众在 3 秒内知道这份 deck 的态度和价值，比单独一个标题信息量大得多。

**为什么 summary 用打勾卡片而非重复正文**：总结页是观众唯一会拍照带走的页。
它该是"结论"不是"回顾"——给 3 条可执行/可记忆的判断，不是把前面再说一遍。

先列提纲再写 spec，就是为了先把这条线理顺。跳过提纲，你写出的必然是
一页页互不相关的 bullets。

## 2 · 信息密度：一页一个论点

每个 `content` 页只承载**一个论点**。判断标准：这页的标题能不能用一句陈述句
说清？如果标题要写成"市场规模、竞争格局与未来趋势"，那是三页的内容塞进了一页。

- 一个数据故事 → 一页（stat_callout / kpi_cards / bars）
- 一组并列要素 → 一页（icon_rows / grid）
- 一次对比 → 一页（two_col / comparison_table）

**宁可多一页，不要挤一页。** 8 页讲清 5 件事，好过 5 页讲糊 8 件事。

## 3 · 节奏：每 3–4 页一个喘息页

连续的高密度信息页会让观众疲劳、走神。每 3–4 页插一个低密度高冲击页：

- `quote` —— 一句金句，把刚才的数据拔高到判断
- `big_number` —— 一个超大数字，制造记忆锚点
- 低密度 `section` —— 顺便起到分段作用

**为什么**：节奏对比制造记忆点。全程满页 = 全程无重点。喘息页是"请记住这个"的
视觉信号。这也是 `meta.layout_warnings` 会盯着"连续纯 bullets"报警的原因——
那是节奏崩塌的信号。

## 4 · 视觉层级：让重要的东西看起来重要

- **kicker（眉标）**：标题上方的小号大写提示标签（如 `"kicker":"市场分析"`）。
  几乎零成本，但瞬间提升专业感——它给标题一个"语境帽子"，让页面有出版物质感。
  cover / section / content / summary 都支持，建议主体页普遍加。
- **数字要大**：stat/kpi 类版式的存在意义就是把数字放大。别把关键数字埋进
  bullets 的一行里——那等于没强调。
- **少即是多**：引擎对每页阴影有预算上限（克制 = 高级）。不要试图在一页里
  堆所有效果。一页一个视觉焦点。

## 5 · 配色与风格：克制是专业的底色

详见 `themes.md`（24 套命名主题完整目录）。**默认通用风（`swiss_klein`，白底）适配
95% 的需求**——没风格诉求就用它。内容是技术/产品/方案/路演或要深色高对比 → `navy_gold`；
用户给了参考图或有强场景属性（中式/财经/学术/报刊/玻璃…）→ 从扩展目录挑一个 `theme` 名。
别因话题"高科技"就默认上深色——拿不准用默认。无论选哪套，**克制**仍是底色。

**色彩主导性**：不管哪个盘子，一份 deck 里要有一个色占 60–70% 的视觉重量
（通常是底色或主色），1–2 个色辅助，**一个**强调色只用在最该被看见的地方
（关键数字、CTA、当前章节）。四个色平均用力 = 没有重点 = 业余。需要同色系
的深浅层次时，把主色调亮/调暗，不要引入新色。

**深浅三明治**：封面 + 结尾页深色、中间内容页浅色，是最稳的结构——深色页
天然有"起"和"收"的仪式感；或者整份 deck 全程深色走高级感。最忌讳的是深浅
随机跳，观众的眼睛会累。

## 6 · 视觉母题：选一个元素，让它贯穿全篇

一份 deck 显得"设计过"而不是"拼出来"，关键往往是**一个反复出现的视觉母题**。
开工前定一个、且只定一个：

- 一个强调色圆点（放在每页标题前 / 章节号旁）
- 图标统一装在同色圆圈里
- 卡片统一用单边粗边条（accent 色，只压一条边）
- 图片统一圆角 / 统一加同款细边框

选定后**每一页都带上它**。母题是"这些页属于同一份 deck"的视觉签名。
不要每页换花样——那不是丰富，是凌乱。`build` 引擎的版式已内置了各自的母题
处理；走 `freeform` 时这条要你自己守。

## 7 · 每页都要有一个视觉元素

纯文字页（标题 + 一堆 bullets）是最容易被忘记的页。**每一页都该有点可看的**
东西——一张图、一个图表、一组图标、一个被放大的数字、一块色卡、一个形状分区。

这正是富版式（`stat_callout` / `kpi_cards` / `icon_rows` / `timeline` …）
存在的意义：它们逼着每页有视觉结构。连续纯 `bullets` 之所以被 `layout_warnings`
盯着报警，就是因为它违反这一条。`single`（纯要点）是**最后手段**，不是默认。

## 图标语义（icon_rows 等版式的 `icon` 字段）

`icon` 填**语义概念名**，引擎自动映射到 Tabler 矢量图标（MIT 授权），
比 emoji/字符专业得多。常用概念名：

`chart` 数据图表 · `users` 用户/团队 · `shield` 安全/合规 · `rocket` 增长/发布 ·
`idea` 创新/洞察 · `security` 安全 · `finance` 财务/资金 · `eco` 环保/可持续 ·
`cpu` 算力/技术 · `target` 目标 · `clock` 时间/效率 · `globe` 全球/网络 ·
`settings` 配置/运维 · `check` 完成/合规 · `trending` 趋势 · `database` 数据/存储

图标库已扩到 ~80 个，**产业 / 政务 / 民生**概念现在都有专属矢量（无需再凑近似）：

- 产业制造：`factory` 工厂 · `logistics`/`truck` 物流 · `供应链` · `warehouse` 仓储 ·
  `robot` 机器人 · `tools` 工具/运维 · `server` 服务器 · `code` 研发/代码 · `通信`/`5g` 通信
- 双碳能源：`solar` 光伏 · `风电` · `battery` 储能 · `recycle` 回收 · `water` 水务
- 政务民生：`公章`/`seal` · `gavel` 法律/执法 · `certificate` 证书资质 · `hospital` 医院 ·
  `school` 学校/教育 · `handshake` 签约/合作 · `fingerprint` 身份认证
- 金融消费：`bank` 银行 · `wallet` 钱包 · `coins` 收益 · `shopping` 消费/零售 · `gift` 福利/权益
- 通用：`mobile` 手机/App · `key` 钥匙/授权 · `puzzle` 协同/集成 · `book` 知识 ·
  `microscope` 科研检测 · `atom` 基础研究 · `plant` 农业 · `car`/`plane`/`ship` 交通 ·
  `trophy` 荣誉 · `presentation` 汇报 · `qrcode` 二维码 · `phone-call` 电话

中英文概念名都能识别（`工厂`=`factory`），引擎也会从 title/desc 文本**自动推断**图标。

填不准时，用 `glyph` 给一个 unicode 字符（`◆ ● ▲ ★ ◭ ◔`）也行，引擎会按
style 渲染成统一的图标徽章。**不要**在 title/desc 里塞 emoji——交给 icon 字段。

## 插图 slot 选择（image 字段）

有图（图表、产品图、架构图、用户上传图）时：

- 数据图表配叙述 → `slot:"right"`，左侧 2–3 条 bullets 解读图，右侧放图。
  这是"图 + 解读"的黄金组合，远胜光放一张图。
- 架构/流程示意，需要大 → `slot:"below_title"`（图占满正文区）。
- 封面/发布会视觉冲击 → `slot:"hero"` 或 `slot:"full"`。
- 精确控制 → 给 `{x,y,w,h}`（英寸，画布 16:9 = 10″×5.625″）。

永远给 `caption`（数据来源/说明）——没出处的图在正式汇报里是减分项。

## 避免清单 —— 这些一眼就是"AI 套模板"

成片显不显业余，很多时候不在于做了什么，而在于**没犯这些错**：

- **绝不在标题下加装饰线 / 下划线 / accent 横线。** 这是 AI 生成 PPT 的
  头号特征。要区隔标题和正文，用留白、用字号差、用底色块——不要画线。
- **正文不要居中。** 段落、列表、bullets 一律左对齐；只有标题、封面大字、
  单个英雄数字才居中。居中的正文很难扫读。
- **字号对比要拉开。** 标题 ≥ 28–36pt，正文 14–16pt，caption 10–12pt。
  标题只比正文大一两号 = 层级塌掉。
- **不要每页同一个版式。** 跨 deck 至少混 3 种；连续 >2 页相同结构就该警惕。
- **不要只精修一页、其余摆烂。** 要么全程到位，要么全程从简——风格不一致
  比朴素更糟。
- **不要纯文字页**（见原则 7）。
- **不要低对比。** 浅字配浅底、深字配深底、深图标配深底都不行——文字和图标
  都要和背景有明确反差。
- **间距要统一。** 卡片间距、留白选定 0.3″ 或 0.5″ 就全程一致，不要东宽西窄。

## 字体

标题用一款**有个性**的字体，正文用一款**干净**的字体，两者配对而非全篇一种。
中文标题可用偏粗的黑体（思源黑体 Heavy / 微软雅黑 Bold），正文用常规黑体
（思源黑体 Regular / 苹方）；西文标题可配 Georgia / Arial Black，正文配
Calibri / Arial。`build` 路径在 spec 顶层加 `"fonts":{"header":"…","body":"…"}`
覆盖；`freeform` 路径在 `addText` 里用 `fontFace`。一份 deck 最多两款字体。

## 自检：交付前问自己

1. 把每页标题抄下来连读，是不是一个连贯的故事？（叙事弧）
2. 有没有哪页标题需要用"和/与/及"连接多个论点？（密度超载 → 拆页）
3. 最长的连续高密度段有几页？超过 4 页没喘息页吗？（节奏）
4. 关键数字是被放大了，还是埋在 bullets 里？（层级）
5. 风格选对了吗？默认通用风 theme/pack 都不传；只有技术诉求才用 navy_gold + dark_gold。（克制 + 主导色）
6. 定了一个视觉母题、且每页都带上了吗？（母题）
7. 有没有纯文字页？每页都有一个视觉元素吗？（视觉元素）

都过了再 build；build 完用 `qa-and-edit.md` 的闭环验。
