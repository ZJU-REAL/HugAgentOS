# 场景：工作台 / 门户首页

> 用户进入应用的第一屏，「看一眼状态 + 快速跳去干活」。不是营销落地页，不要大 Hero 图套路。

## 场景定位

- **用户目标**：进来 3 秒内知道「有什么要我处理的 / 关键数据怎么样 / 我常用的入口在哪」。
- **不是**：品牌展示页、数据大屏（那是 dashboard）、某个对象的详情（那是 detail）。
- **典型业务**：部门门户、系统首页、我的工作台、班组看板首页。

## 布局骨架 / 信息架构

自上而下的纵向叙事（移动优先）：

```
┌─────────────────────────────────────┐
│ 欢迎区（问候 + 当前用户 + 日期/待办数）│  ← 克制，1 行 2 行，不做大图 banner
├─────────────────────────────────────┤
│ 核心指标（3-5 个 KPI，主次分明）      │  ← 门户级概览，不是完整看板
├─────────────────────────────────────┤
│ 快捷入口（分组，有主次）              │  ← 高频操作在前，长尾折叠/靠后
├─────────────────────────────────────┤
│ 我的待办 / 最近动态                    │  ← 列表，可点进详情
└─────────────────────────────────────┘
```

PC 端可将「核心指标 + 快捷入口」并成上区、「待办 + 动态」并成下区双栏；移动端全部纵向堆叠。

## 信息密度

**中**。门户是概览不是明细——每块只放最重要的，深入靠点击跳转。留白要够，别把首页做成信息轰炸。

## 焦点（hero）元素

**核心指标 + 主快捷入口**。欢迎区是配角，不要让问候语或装饰图占掉首屏焦点。

## 组件套餐（映射 design-system.md / component-jsx-guide.md）

- 指标卡：`design-system.md` 卡片 + 大字号数字（`tabular-nums`）+ 次要标签
- 快捷入口：图标 + 文字的入口块（图标此处是功能性导航，允许，但风格统一）
- 待办/动态：`design-system.md` 数据列表行（listItem）
- 空态：`design-system.md` 空状态（新用户无待办时友好提示 + 主操作）

## 本场景专属去 AI 味要点

- ❌ **禁营销大 Hero**：不要顶部一张巨图 + 渐变遮罩 + 大标语。B 端门户不是官网。
- ❌ **禁快捷入口平铺一片等宽图标格子**：入口要分组、有主次（高频 vs 低频），不是 4×N 的对称九宫格。
- ✅ **欢迎区克制**：一句问候 + 关键数字（如「今天有 3 项待办」）即可，别配大插画和 emoji。
- ✅ **指标有主次**：最重要的 1 个指标可放大/置于视觉起点，其余次级，不要 5 个 KPI 一样大平铺。

## 视觉方向决策块示例

```markdown
### 【视觉方向决策】
- 页面类型：workbench（用户诉求是"部门门户首页，进来先看待办和常用功能"）
- 气质关键词：稳重、高效、克制
- 项目特定原则：① 待办数量必须首屏可见；② 高频入口（提交申请/查进度）永远在入口区最前；③ 指标只放部门最关心的 3 个
- 布局骨架：欢迎区(1 行) → 3 指标(主次) → 快捷入口(2 组) → 我的待办列表
- 信息密度：中（概览为主，深入靠跳转）
- 视觉焦点：核心指标 + 主快捷入口

### 【差异化 5 维】
1. 辅助/强调色：待办数用点缀色强调，逾期用告警红（仅此处）
2. 中性冷暖偏色：冷灰基底（政企门户，专业感）
3. 圆角性格：微圆（克制企业感，全页统一）
4. 排版性格：指标数字大字 + tabular-nums；标题 600 正文 400
5. 装饰母题：重点待办左侧主色竖条 + 单色阶分区（不用彩色卡片区分）

### 【反默认说明】
用冷灰基底 + 微圆 + 单色阶分区替代「灰白底 + 8px 圆角彩色卡片网格」，且快捷入口分组而非九宫格平铺——3 维偏离默认脸。

### 【图标策略】
内联 SVG 语义集，细描边；仅快捷入口和状态用图标，标题不配装饰图标。
```
