# 场景：官网 / 落地页 / 品牌展示页

> 面向外部传播或内部宣导的「讲清楚价值 + 建立信任 + 引导下一步」。它不是 B 端工作台，也不是纯详情页；信息密度可以低一些，但每屏都要有明确叙事任务和视觉锚点。

## 场景定位

- **用户目标**：快速理解「这是什么 / 对我有什么价值 / 为什么可信 / 下一步做什么」。
- **不是**：工作入口首页（那是 workbench）、数据驾驶舱（那是 dashboard）、单对象规格详情（那是 detail）。
- **典型业务**：产品官网、活动页、招商页、方案介绍页、组织内推广页、公开分享页。

## 布局骨架 / 信息架构

先写清页面是单页还是多页。宜搭自定义页通常先做单页；如果内容天然分层很深，再用页面内导航壳或多个自定义页承载。

```
┌─────────────────────────────────────┐
│ Hero：品牌/产品名 + 唯一主张 + 主动作  │  ← 只讲一个核心承诺
├─────────────────────────────────────┤
│ 价值证明：3 个以内的核心能力/收益       │  ← 不做平均卡片堆砌
├─────────────────────────────────────┤
│ 场景/流程：用图示或错位结构讲使用路径    │
├─────────────────────────────────────┤
│ 信任背书：数据、案例、资质、客户声音      │
├─────────────────────────────────────┤
│ CTA：下一步动作（咨询/试用/查看表单）     │
└─────────────────────────────────────┘
```

移动端：Hero 不要做满屏过高，首屏底部要露出下一段内容；复杂视觉锚点改为单列或横向滚动，不压住主文案。

## 信息密度

**低-中**。landing 讲的是价值路径，不是把所有功能塞满。每个 section 只承担一个任务：吸引、解释、证明、转化。长文本要转成图示、对比、数据卡、流程或引文。

## 焦点（hero）元素

**品牌/产品主张 + 一个视觉锚点**。Hero 的 H1 应是品牌名、产品名或具体 offer；价值解释放在副标题，不把空泛口号当主标题。

## Section 构图策略

每个 section 必须选不同构图，禁止连续复用同一种布局。可选手法：

| 构图手法 | 适用 |
|---|---|
| 文字主导 | 理念/服务型产品，缺少强图片素材时 |
| 不对称分割 | B2B/SaaS/企业服务，信息层级要清晰时 |
| 网格打破 | 多能力展示，但需要主次而不是三卡片平均铺平 |
| 重叠与破界 | 创意/视觉强品牌，有真实产品图或场景图时 |
| 时间线/蛇形 | 发展历程、流程、服务路径 |
| 超大留白对比 | 高端、咨询、法律、金融等克制调性 |
| 数据证明区 | 有真实数字/案例时，用大数字 + 短解释建立信任 |

Hero 构图按品牌调性选，不默认左文右图，也不默认满屏背景。没有真实图片时，优先用文字主导、数据图示、流程图，而不是编造图片 URL。

## 素材与真实性

- Logo、产品图、客户案例、联系方式、社媒链接、数据背书必须来自用户提供或已验证资料；没有就留空/隐藏，不编造。
- 每个 section 至少有一个视觉锚点：真实图片、产品截图、流程图、数据图示、引文排版、对比表或大字排版。
- 图片/视频素材只走已验证 URL 或合规 CDN；不要用随机占位图替代真实业务素材。

## 组件套餐（映射 design-system.md / canvas-design-system.md）

- Hero：大标题 + 副标题 + 主/次操作；导航隐藏时可叠加 `yida-nav-shell` 顶部导航。
- 价值证明：非等宽平铺的能力区，主能力放大，其余弱化。
- 场景流程：步骤条、时间线、错位图文、数据图示。
- 信任背书：案例卡、客户引文、数据证明、资质列表。
- CTA：单一主动作，跳原生表单/外链/详情页；线索收集不要手写表单。

## 本场景专属去 AI 味要点

- ❌ **禁万能营销文案**：「一站式赋能」「极致体验」「智能高效」这类空话必须改成具体业务承诺。
- ❌ **禁连续同构 section**：左文右图 + 三卡片 + 左文右图一路复制，是最明显的模板脸。
- ❌ **禁无素材硬做大 Hero 图**：没有真实素材时用文字排版/数据图示/流程图，不编造图。
- ✅ **每屏一个任务**：吸引、解释、证明、转化分清楚，不让一个 section 同时承担所有内容。
- ✅ **CTA 克制明确**：全页主动作一致，按钮文案说清下一步，不堆多个同权按钮。

## 视觉方向决策块示例

```markdown
### 【视觉方向决策】
- 页面类型：landing（用户诉求是"做一个产品介绍官网页，对外讲清方案价值"）
- 气质关键词：可信、清透、专业
- 项目特定原则：① 首屏只讲"审批效率提升"一个主张；② 没有真实客户 Logo 时不做客户墙；③ CTA 统一跳到原生咨询表单
- 布局骨架：Hero(产品名+唯一主张) → 价值证明(1 主 2 次) → 使用流程(时间线) → 数据背书 → CTA
- 信息密度：低-中（每屏一个叙事任务，避免功能堆砌）
- 视觉焦点：产品主张 + 流程图式视觉锚点
- 场景专项策略：Hero 用文字主导 + 右侧流程图示；价值区用网格打破；流程区用时间线；背书区用数据证明；CTA 跳原生表单

### 【差异化 5 维】
1. 辅助/强调色：用低饱和青绿做证明数据点缀，告警/错误色不参与品牌表达
2. 中性冷暖偏色：中性灰偏暖，降低工具感
3. 圆角性格：标准圆，保持现代但不消费化
4. 排版性格：Hero 大标题强字重，正文短句化；数据证明用 tabular-nums
5. 装饰母题：细线流程节点 + 单色数据证明块 + 非对称 section 节奏

### 【反默认说明】
不是统一卡片网格，而是文字主导 Hero、错位价值区、时间线流程三种构图轮换；中性偏暖和细线流程母题也偏离默认灰白蓝模板。

### 【图标策略】
内联 SVG，仅用于流程节点和 CTA 箭头；section 标题不配装饰图标。
```
