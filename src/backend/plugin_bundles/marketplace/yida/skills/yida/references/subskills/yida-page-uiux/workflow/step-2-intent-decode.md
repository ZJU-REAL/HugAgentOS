# Step 2：意图解码

> 从需求里提炼这个页面的**气质**，为差异化提供依据。这是 Step 4 差异化决策和 Step 6 自检的溯源根据。

- **2-3 个气质关键词**：如「稳重可信 / 轻盈现代 / 克制高端 / 温暖亲和 / 前沿科技」。
- **3-5 条项目特定设计原则**：结合业务写具体约束，而非套话。例：
  - 「进销存看板 → 库存预警必须一眼可见（红色只留给告警）」
  - 「财务门户 → 金额用 tabular-nums 对齐，禁跳动」
  - 「品牌落地页 → 首屏只讲一个主张，后续每个 section 都有不同构图和一个视觉锚点」
  - 「经营驾驶舱 → KPI/图表/表格的所有数值必须带单位、量级和业务含义」

避免写「简洁 / 现代 / 美观」这类放到任何页面都成立的空话——那样 Step 6 的「替换测试」会直接判不合格。

## 场景专项提炼

- **landing/官网类**：额外提炼「唯一主张」「受众」「转化动作」「可用素材」。没有真实联系方式、Logo、产品图、客户案例时，明确标为缺失，后续落地不能编造。
- **dashboard/看板类**：额外提炼「核心判断问题」「Shell 复杂度」「页面原型」「关键指标口径」。没有口径时先用业务假设占位，并在决策块标注。

## 产出

在决策块记录「气质关键词」（2-3 个）+「项目特定设计原则」（3-5 条，具体到业务）；landing/dashboard 额外记录场景专项策略。
