# 场景：详情 / 展示页

> 围绕**单个对象**做深度呈现的页面。信息不多但要「讲清楚、有质感」，最容易被做成 AI 味模板脸——本场景**叠加详情页专属纪律**。

## 场景定位

- **用户目标**：深入了解某一个对象的全貌（一个订单、一个商品、一个客户、一份报告）。
- **不是**：一堆对象的列表（那是 list）、聚合指标（那是 dashboard）。
- **典型业务**：订单详情、商品详情、客户档案、项目详情、报告展示页。

## 布局骨架 / 信息架构

单对象的**叙事编排**——不是字段的机械罗列，而是「先看什么、再看什么」的顺序设计：

```
┌─────────────────────────────────────┐
│ 主区（hero）：对象核心身份 + 主图/主数据 │  ← 完整展示，不裁切、不发光
├─────────────────────────────────────┤
│ 关键信息组（分章节，非平铺字段墙）     │  ← 按业务逻辑分组，有主次
├──────────────────┬──────────────────┤
│ 主要内容（叙事）  │ 侧栏（元信息/操作） │  ← 非对称，主 70% 次 30%
├──────────────────┴──────────────────┤
│ 关联对象 / 时间线 / 附件               │
└─────────────────────────────────────┘
```

移动端：全部纵向堆叠，主区在最前，侧栏元信息下沉。

## 信息密度

**低—中**。详情页是「少而精」——单对象信息本就不多，靠**留白、字号跨度、分组**造质感，别硬塞满或堆装饰凑版面。

## 焦点（hero）元素

**对象的核心身份 + 主图/主数据**。用户进来第一眼要确认「这就是我要找的那个对象」，主区必须完整、突出、无干扰。

## 组件套餐（映射 design-system.md / component-jsx-guide.md）

- 主区：大字号对象名（标题与正文字号差 **≥2.5×**）+ 主图完整展示 / 核心数据大字
- 章节：`design-system.md` 卡片分组，章节标题**具体到该对象**（不是「基本信息」这种万能词）
- 字段：标签-值成对，弱化标签、加重值；金额/数量 `tabular-nums`
- 时间线/关联：`design-system.md` 列表行 / 时间线
- 操作：主操作按钮突出，次要操作弱化；需录入走 `iframe workbench/{formUuid}?iframe=true`

## 本场景专属去 AI 味要点（叠加详情页纪律）

- ✅ **主图/主数据完整展示**：hero 对象不裁切、不加彩色发光、不套渐变遮罩；用留白和对齐凸显，不用特效。
- ✅ **≥2 种非发光装饰手法**：如左侧主色竖条、章节起始细线、超大字号视觉锤、单色分区、非对称双栏——**禁彩色发光/半透明彩色 blob/渐变文字**。
- ✅ **反模板叙事顺序**：不要「标题→基本信息→详细信息→附件」这套万能模板顺序；按**这个对象**最该先讲的逻辑排（如商品先讲卖点/主图，订单先讲状态/金额）。
- ✅ **字号跨度拉开**：主标题与正文 **≥2.5×**，制造视觉层级，别全页一个字号靠加粗硬撑。
- ✅ **章节标题具体化**：用「本单发货记录」而非「物流信息」，用「客户履约情况」而非「其他信息」——标题带对象语境。
- ❌ **禁字段墙**：一屏平铺 20 个 label:value = 没消化过的数据库导出。按业务分组、有主次、隐藏低频字段。
- ❌ **禁每章节标题配一枚装饰图标**：这是最典型详情页 AI 味。图标只在功能处（操作/状态），标题纯文字。
- ❌ **禁 AI 万能文案 / emoji**：不写「让我们一起了解…」「✨核心亮点」这类填充话术。

## 视觉方向决策块示例

```markdown
### 【视觉方向决策】
- 页面类型：detail（用户诉求是"商品详情页，展示单个商品的全貌"）
- 气质关键词：精致、有质感、克制
- 项目特定原则：① 主图必须完整展示不裁切；② 卖点/规格先于流水信息；③ 价格用大字 tabular-nums 当视觉锤
- 布局骨架（反模板叙事）：主图+品名+价格(hero) → 核心卖点(3点) → 规格参数(分组) → 主内容左70%(详情描述) + 右30%(库存/操作) → 关联商品
- 信息密度：低-中（留白 + 字号跨度造质感）
- 视觉焦点：主图 + 品名 + 价格

### 【差异化 5 维】
1. 辅助/强调色：价格用点缀色强调（暖金=品质感），规格标签用中性；不满屏上色
2. 中性冷暖偏色：暖灰基底（商品展示的人文/质感气质）
3. 圆角性格：标准圆（现代、精致，主图卡片微圆不喧宾夺主）
4. 排版性格：品名大字 vs 参数小字跨度 ≥2.5×；价格 tabular-nums；大标题微收 letter-spacing
5. 装饰母题：章节起始细线 + 主图完整留白展示 + 非对称双栏（左详情右操作）

### 【反默认说明】
用暖灰 + 字号跨度 + 非对称叙事替代「灰白底 + 均匀字号 + 字段墙 + 每章配图标」，主图完整展示无发光——4 维偏离默认详情脸。

### 【图标策略】
内联 SVG，仅操作按钮（加购/收藏/分享）用图标；章节标题（卖点/规格/详情）纯文字不配装饰图标。
```
